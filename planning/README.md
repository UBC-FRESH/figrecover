# Planning Notes

Use this directory for focused research notes, design records, and contracts
that are too detailed for `ROADMAP.md`.

The canonical long-form scope and phase breakdown is
`package-scope-and-implementation-plan.md`; keep `ROADMAP.md` as the shorter
issue tracker map.

Planning notes should be practical and evidence-oriented:

- state the question being investigated;
- record source files, documents, libraries, or commands inspected;
- separate verified behavior from assumptions;
- capture unresolved risks and follow-up work;
- avoid copying private notes, raw transcripts, source PDFs, rendered pages,
  cropped figures, VLM outputs, or generated bulky artifacts into tracked files.
